# 수학비서 월요일 밤 강의 - 상세 기획서

## 1. 강의 개요

| 항목 | 내용 |
|------|------|
| 프로그램명 | **수학비서 AI 활용법 강의** |
| 시간 | 매주 월요일 밤 22:30 ~ 23:30 (60분) |
| 대상 | 수학 학원 강사, 원장님 |
| 형태 | 온라인 라이브 (Zoom / Google Meet) |
| 목적 | ① AI 활용 역량 제공 ② 수학비서 서비스 간접 홍보 ③ 커뮤니티 형성 |

### 왜 월요일 밤 22:30인가?
- 학원 수업이 끝난 후 여유 시간대
- 한 주의 시작에 새로운 도구/방법을 배워 바로 적용 가능
- 다른 교육 프로그램과 시간대 차별화

## 3. 강의 운영 설계

### 3.1 강의 포맷 (60분 구성)

| 시간 | 내용 | 비고 |
|------|------|------|
| 22:30 ~ 22:35 (5분) | 오프닝 & 지난주 복습 | 출석 체크 자동화 |
| 22:35 ~ 23:00 (25분) | 이론 설명 + 데모 | 화면 공유 |
| 23:00 ~ 23:20 (20분) | 실습 시간 | 실시간 Q&A |
| 23:20 ~ 23:25 (5분) | 수학비서 연계 팁 | 간접 홍보 포인트 |
| 23:25 ~ 23:30 (5분) | 다음주 예고 & 과제 | 후기 작성 안내 |

### 3.2 간접 홍보 전략

**원칙**: 강의 자체가 가치를 제공하되, 자연스럽게 수학비서와 연결

1. **데모 활용**: 실습 예제에 수학비서 기능을 자연스럽게 포함
2. **Before/After**: "직접 만들면 이렇게 → 수학비서 쓰면 이렇게" 비교
3. **얼리어답터 혜택**: 강의 수강자에게 수학비서 베타 우선 초대권 제공
4. **매주 마지막 5분**: "이번 주 배운 내용, 수학비서에서는 이렇게 됩니다" 코너

### 3.3 참가자 관리

| 항목 | 방법 |
|------|------|
| 정원 | 회차당 30~50명 (소통 가능한 규모) |
| 가격 | 무료 (리드 확보 목적) 또는 소액 (5,000원 - 노쇼 방지) |

## 4. 자동화 설계

### 4.1 전체 자동화 흐름

```
[신청] → [확인메일] → [리마인더] → [강의참여] → [후기요청] → [후기수집] → [다음회차안내]
```

### 4.2 신청 자동화

```
참가자가 랜딩페이지에서 신청 폼 제출
    ↓
① Google Form / Tally 에서 데이터 수집
    ↓
② Zapier / Make(Integromat) 트리거 발동
    ↓
③ 자동 처리:
   - Google Sheets에 참가자 정보 기록
   - 확인 이메일 발송 (Zoom 링크 포함)
   - 카카오 알림톡 발송 (선택)
   - Google Calendar 초대 자동 생성
    ↓
④ CRM(Notion DB / Airtable)에 참가자 상태 업데이트
```

**추천 도구 조합**:
| 역할 | 도구 | 비용 |
|------|------|------|
| 폼 수집 | Tally (무료) 또는 Google Forms | 무료 |
| 자동화 연결 | Make (구 Integromat) | 무료~$9/월 |
| 이메일 발송 | Mailchimp 또는 Stibee (한국) | 무료~월 1만원 |
| 알림톡 | 솔라피 또는 알리고 | 건당 8~15원 |
| 참가자 관리 | Notion DB 또는 Google Sheets | 무료 |
| 화상 강의 | Zoom (무료 40분 / Pro $13.3/월) | 무료~$13.3/월 |

### 4.3 리마인더 자동화

| 시점 | 채널 | 내용 |
|------|------|------|
| 신청 직후 | 이메일 + 알림톡 | "신청 완료! 월요일 밤 10:30에 만나요" + Zoom 링크 |
| 강의 3일 전 (금요일) | 이메일 | 이번 주 강의 주제 미리보기 + 준비물 안내 |
| 강의 당일 오전 | 알림톡 | "오늘 밤 10:30 수학비서 AI 강의! 잊지 마세요" |
| 강의 1시간 전 (21:30) | 알림톡 | "1시간 후 시작! [입장 링크]" |

### 4.4 후기 수집 자동화

```
강의 종료 (23:30)
    ↓
① 30분 후 (00:00) - 자동 후기 요청
   채널: 알림톡 + 이메일
   내용: "오늘 강의 어떠셨나요? 30초면 됩니다 [후기 링크]"
   후기 폼: Tally 또는 Google Forms (별점 + 한줄평)
    ↓
② 미작성자 리마인드 (다음날 화요일 12:00)
   채널: 알림톡
   내용: "어제 강의 후기 남겨주시면 다음 강의 자료를 미리 보내드려요!"
   인센티브: 후기 작성 시 추가 자료 or 수학비서 할인 쿠폰
    ↓
③ 후기 수집 완료 → 자동 처리
   - Google Sheets에 후기 데이터 기록
   - 별점 4점 이상 → 랜딩페이지 후기 섹션에 자동 반영 (승인 후)
   - 후기 작성자에게 감사 메일 + 혜택 발송
```

### 4.5 자동화 구현 방법 (Make 시나리오)

**시나리오 1: 신청 처리**
```
Tally 폼 제출 → Make 웹훅
  → Google Sheets 행 추가
  → Stibee 구독자 추가 + 환영 메일
  → 솔라피 알림톡 발송
  → Google Calendar 이벤트 생성
```

**시나리오 2: 리마인더**
```
Make 스케줄 (매주 월 09:00)
  → Google Sheets에서 이번 주 신청자 조회
  → 솔라피 알림톡 일괄 발송
  → 21:30 재발송 (1시간 전 알림)
```

**시나리오 3: 후기 수집**
```
Make 스케줄 (매주 화 00:00)
  → 솔라피 후기 요청 알림톡 발송
  → 화 12:00 미작성자 리마인드 발송

Tally 후기 폼 제출 → Make 웹훅
  → Google Sheets 후기 기록
  → 별점 4+ 시 랜딩페이지 후기 풀에 추가
  → 감사 메일 + 혜택 쿠폰 발송
```
